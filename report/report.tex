\section*{\LARGE Introduction}
Reinforcement learning is an artificial intelligence approach that uses agents set to take actions in complex environments and learn from their interactions. The objective is to maximise the expected cumulative reward.

As defined by Sutton \& Barto, 2018, Ch.2, Sec 2.1, Multi-armed bandits (MABs) are a simplified reinforcement learning setting where an agent repeatedly chooses among several bandits to maximise reward under uncertainty. 

A multi-armed bandit problems can be interpreted as a casino parlor: multiple slot machines, which are referred to as bandits, are set to give random rewards. The value of these rewards and their likelihood can be hardcoded to behave like a probability density function. I.E. the slot machines, when pulled, may give positive or negative rewards that converge around a mean. That said, many kinds of statistical distributions may be implemented, such as uniform distributions with upper and lower bounds, triangular distributions with a mode, and so on.

Using reinforcement learning, agents must be set with some kind of policy that enables taking the optimal choice among the multiple slot machines, I.E. the bandits, available. Sequentially pulling on different bandits, will influence the agents' decisions by providing evaluative feedback.

A central difficulty is the exploration exploitation trade-off. When action value estimates are maintained, the arm with the highest current estimate is the greedy choice. Selecting it is exploitation because it uses the best option according to current knowledge, while selecting a non-greedy arm is exploration because it can reduce uncertainty and improve future estimates. Exploitation tends to maximise expected reward in the next step, but exploration can lead to higher total reward over a longer horizon by helping the agent discover arms that are actually better. Since a single action choice cannot do both at once, balancing exploration and exploitation is intrinsical in bandit problems.

A common way to implement action value methods is to estimate each arm's value from observed rewards. One simple estimator is the sample average, computed as the total reward collected from an arm divided by the number of times it has been selected. (Antonacopoulos et al., 2025)

To compare bandit methods, performance is often visualised using reward based curves such as cumulative or average reward over time, together with the frequency of selecting the optimal arm. (Antonacopoulos et al., 2025)

In many real applications, reward distributions are non-stationary and can change over time. This motivates methods that adapt to recent observations rather than relying on full historical averages. Recent work has also explored explicit uncertainty measures, for example using the fractal dimension of reward history and integrating it into standard bandit algorithms, with reported improvements in cumulative reward in non-stationary settings. (Nandhini \& Pandian, 2025)

To approach the problem, a set of agents was formulated. In Table \ref{table_1} the designed agents can be seen in terms of their estimator, strategy (I.E. its policy) and which environment it faced. The two environments were designed as follows:

\begin{itemize}
  \item A static environment consisting of many bandits behaving according to a mixture of statistical distributions with fixed parameters. This scenario tests the policies Softmax and Espislon-Greedy against each other. Fixed distribution bandits are ideally estimated with average estimators, which are set equally to all the agents.
  \item A dynamic environment consisting of the same bandits, this time with changing parameters. Given that the best performing policy in the previous scenario was Softmax, it was assigned to most of the agents, and different estimators were tested. A single Espislon-Greedy policy agent was kept as reference.    
\end{itemize}

\subsection{Dynamic Environment}

At the start, all methods rarely select the best arm, because their value estimates are uncertain. The average estimator (green) learns reasonably fast but then degrades as the environment changes, since it keeps all past rewards and therefore reacts slowly to non-stationary bandits.

The moving average (yellow) reacts faster to changes than the plain average, so the percentage of best-arm selections improves more steadily. The incremental update with fixed step size (orange) adapts the quickest and maintains the highest fraction of optimal-arm selections over time, showing that constant-step updates are more appropriate in a dynamic environment.

Overall, these results support the observation that in static environments keeping exploration  as low as possible is ideal. This is something that Softmax can easily account for. Whereas in dynamic environments it is necessary to implement estimators that adapt quickly to changes, which also includes Softmax.

\begin{itemize}
  \item Experimental results show the average estimator decays over time because it calculates the value with historical memory
  \item Moving average shows lower efficiency at the start because it has lower information/data to work with, but over time it improves significantly due to the dynamic nature of the estimator 
  \item For the incremental estimator, not only did it start off with relatively high optimal selection, but also it performed the most optimally over time. This is because it accumulates data results, and it has more accumulative data to work with. 
  \item These results indicate that dynamic estimators are superior to the average estimator, and the incremental estimator is the most optimal (within our timespan). Given more time, there is a potential for the moving average to overtake the incremental estimator, but this cannot be accurately shown from the data.
\end{itemize}

\section*{\LARGE Discussion}
\subsection*{Problem 2}

The trade-off between exploration and exploitation is a central challenge in reinforcement learning. To obtain high reward, an agent should prefer actions that have produced good results in the past (exploitation), but to discover such actions it must also try actions that have not been tried often or at all (exploration). (Berger-Tal, Nathan, Meron, \& Saltz, 2014). In other words, the agent must exploit what it already knows to gain immediate reward, while also exploring to improve its knowledge and make better action selections in the future. (Sutton \& Barto, 2014)

In multi-armed bandits, the exploration-exploitation dilemma arises because, at each step, the agent must decide whether to play the arm that has produced the highest average reward so far (exploitation) or to try a different arm to improve its estimate of that arm's true value (exploration). Formally, each arm has an unknown reward distribution, and the agent only learns about these rewards through repeated interaction, which makes this trade-off unavoidable.

The goal of a multi-armed bandit agent is to find an appropriate balance between exploration and exploitation that minimises the regret over a given time horizon. Regret is defined as the difference between the cumulative reward that would have been obtained by always selecting the optimal arm and the cumulative reward obtained by the agent's strategy. A good bandit algorithm aims for regret that grows sublinearly with time, so that the average regret per step converges to zero. (Bubeck \& Cesa-Bianchi, 2012)

One simple strategy to address this trade-off is the epsilon-greedy method. In epsilon-greedy, at each step the agent chooses a random arm with probability $\varepsilon $ (exploration) and chooses the arm with the highest current estimated value with probability  $1-\varepsilon $ (exploitation). The parameter $\varepsilon $ therefore controls the balance between exploration and exploitation: larger values of $\varepsilon $ encourage more exploration, while smaller values favour exploiting the best-known arm. (Sutton \& Barto, 2014)

\subsection*{Problem 3}

The true value of an action is the average reward that was received after that action was taken such that the difference between methods lies in how they use that uncertainty to balance exploration and exploitation.

\bigskip

In a more strict mathematical sense:

\begin{center}
  $Q^{\pi}_{s, a} = E_{\pi}\{ R_{t} | s_{t}=s, a_{t} = a \}$ \\
  \bigskip

  \textit{The action value with policy $\pi$, given the state space $s$ and action space $a$, is equal to the estimated value of the reward $R$ at the current time $t$ given that the agent has taken action $a_{t}$ under the state $s_{t}$.} \\ 
  \bigskip

  \textit{Which can be calculated as:}
  \bigskip

  $E_{\pi}\{ \sum_{k = 0}^{\infty} \gamma^{k}r_{t+k+1} | s_{t}=s, a_{t} = a  \} $

\end{center}

This principle may be adapted based on the kind of estimator and policy of an agent.

\subsubsection*{Incremental Update Rule}

It uses a fixed step size $\alpha $, which makes recent rewards more important than old ones. This was the best estimator for the tested dynamic environment. It adapted the fastest to reward distribution changes and kept the highest percentage of optimal arm selections over time. It also started well from the beginning, not only at the end. The team notes that moving average could eventually overcome it given more time, but this was not confirmed by the data. Mathematicaly described with the expression:

\bigskip

\begin{center}
  $Q_{n+1} = Q_n + \alpha [R_n - Q_n]$
\end{center}
\subsubsection*{Sample-Average Estimator}

This method estimates each arm's value by calculating the mean of all rewards received from that arm.
In static environments it works well, especially when consistent with small epsilon values. The agent does not need much exploration once it learns a good approximation of the bandits' expected reward. However, in dynamic environments this estimator becomes a problem because it gives equal weight to old and new rewards, so it cannot adapt quickly with the environment changes. Formally defined as:

\bigskip

\begin{center}

  $Q_n(a) = \frac{1}{n} \sum_{i=1}^{n} R_i$
  
\end{center}

\subsubsection*{Moving Average Estimator}

Instead of using all past single rewards, this method only considers the last N rewards (windows of 100 and 20 were used).
At the beginning, it performs worse than the total average estimator because it has less data. But in dynamic environments, it improves over time, since it forgets old rewards and reacts better to changes. A window of 20 showed slightly better results than a window of 100 in most cases. Formally defined as:

\bigskip

\begin{center}
$  Q_n(a) = \frac{\sum_{i=\max(1, n-W+1)}^{n} R_i}{\min(n, W)}$ 

\bigskip

$\text{Where } Q_n(a) =_{\text{def}} \text{the sum of the rewards from pull max } (1, n-W+1) \text{to } \frac{n}{/min(n, W)}$
\end{center}

\subsubsection*{$\varepsilon $-Greedy Strategy}

The agent exploits the best known arm with probability $1-\varepsilon $ and explores randomly with probability $\varepsilon $ where, as set for this study, $\varepsilon \in [0.1, 0.3, 0.5, 0.7, 0.9]$.

In static environments, low epsilon (0.1, -0.3) was clearly better because excessive exploration hurts performance when arm values do not change. In dynamic environments, moderate exploration was more useful because the agent needs to detect when the best arm changes. Initially, single runs were used with different random seeds. Results varied between seeds, which shows the stochastic nature of the problem. To account for this, the final experiments were performed in groups of 50 runs whose results were averaged. Looking at the conifidence intervals is a good to show the most likely space of single scenarios.

\subsubsection*{Softmax Strategy}

Instead of a binary explore/exploit decision, Softmax assigns probabilities to each arm based on learned preferences, updated by gradient ascent. Thus, its function $\sigma(z)$ takes in a mathematical tuple made out of a k amount of estimations ($z_{1} \ldots z_{k}$), and assigns them a value $\in (0, 1)$. I.E. $\mathbb{R}^{K} \rightarrow (0\text{,}1)^{k}$; where $\sigma(z)$ is defined as:

\bigskip

\begin{center}
  $\sigma(z)_{i} = \frac{e^{z_{i}}}{\sum_{j = 1}^{K} e^{z_{j}} }$
\end{center}

\bigskip

This implies that every input will be applied to the standard exponential function, and then normalised by the sum of every of such exponents.

The preference $ - \beta \text{ for } \beta \in \mathbb{R}$ may be treated as a parameter that can be adjusted to either concentrate the parameters on the best estimations ($\beta$ > 1), or spread them around lesser estimations (0 < $\beta$ < 1). This results in the equation:

\bigskip

\begin{center}
  $\sigma(z)_{i} = \frac{e^{- \beta z_{i}}}{\sum_{j = 1}^{K} e^{- \beta z_{j}} }$
\end{center}

\bigskip

The code used for this study manages this parameter conceptually as a temperature, which is but the reciprocal: $\beta$ = 1/$T$. 

\subsubsection*{Considerations on Action Value Methods}

Softmax was the best strategy overall for static environments. It concentrated the probability on the best arm more raidly than epsilon-greedy, while still doing some exploration. And so, achieved the highest optimal-arm selection rate among all strategies tested.

In general, static environments need low exploration and stable estimators, while dynamic environments need methods that can forget old data and keep exploring. The incremental update is the most suitable estimator for non-stationary settings.

\section*{\LARGE Conclusion}
It was found that the Softmax strategy performed the most optimally within this particular setup; in contrast with Epsilon-greedy. The subsequent experiment tested for an optimal estimator with a group of Softmax policy agents. It was found that when faced with responsive bandits, the best estimator was incremental update, followed by moving average with a window of 20. The confidence intervals show that Incremental Update has a quicker response, and occasionally greatly outperforms the others. Also, the confidence intervals for the Moving Average estimator show that it can be less consistent and has less potential overall.
